# Project Json Security Policy

## Supported Versions

Project Json is currently under active development. Security updates and fixes will be applied to the latest development version.

| Version | Supported |
|--------|-----------|
| 1.x (development) | ✅ |
| 0.x (experimental builds) | ⚠️ limited |
| < 0.1 | ❌ |

Because Project Json is an experimental cybersecurity research project, older builds may not receive security updates.

---

## Security Philosophy

Project Json is developed using a **security-first engineering approach**.  
The project explores how AI-assisted systems can support cybersecurity workflows such as:

- threat analysis
- log investigation
- anomaly detection
- digital forensics automation

Security considerations are integrated directly into development, including:

- defensive programming
- input validation
- controlled handling of investigation artifacts
- avoidance of sensitive data in the repository

---

## Reporting a Vulnerability

If you discover a security vulnerability in Project Json, please report it responsibly.

**Preferred contact method:**

Open a private GitHub issue or contact the maintainer.

Include the following information when reporting a vulnerability:

- description of the vulnerability
- steps to reproduce the issue
- affected file or module
- potential impact
- suggested mitigation if known

---

## Responsible Disclosure

Please do **not publicly disclose vulnerabilities** until they have been reviewed and addressed. This helps protect users and prevents potential abuse.

All legitimate reports will be acknowledged and reviewed.

---

## Scope

This policy applies to all components within the Project Json repository, including:

- AI reasoning modules
- cybersecurity automation tools
- log analysis utilities
- investigation scripts
